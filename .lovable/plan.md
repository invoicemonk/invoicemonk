

# Send Receipt via Email

## Overview
Add the ability for users to send receipts via email, mirroring the existing invoice email flow. This requires a new edge function, a send dialog component, and a "Send" button on the receipt detail page.

## Changes

### 1. New Edge Function: `supabase/functions/send-receipt-email/index.ts`
- Mirrors the `send-invoice-email` pattern: auth, validation, PDF generation, Brevo email, audit log
- Accepts `{ receipt_id, recipient_email, custom_message? }`
- Fetches the receipt (RLS-enforced), reuses `generate-receipt-pdf` logic inline (or calls pdfmake directly) to generate PDF base64
- Builds an HTML email body with:
  - Business header (logo + name)
  - "Dear [payer name]" greeting + custom message
  - Receipt summary box (receipt number, amount, payment date, method)
  - Link to public verification page
  - QR code for verification
  - PDF attachment notice
  - Business contact footer + Invoicemonk footer
- Sends via Brevo API (same `BREVO_API_KEY` + `SMTP_FROM` secrets already configured)
- Logs `RECEIPT_SENT` audit event + creates notification

### 2. Update `supabase/config.toml`
- Add `[functions.send-receipt-email]` with `verify_jwt = false` (auth validated in code)

### 3. New Component: `src/components/receipts/SendReceiptDialog.tsx`
- Mirrors `SendInvoiceDialog` structure
- Props: `open`, `onOpenChange`, `receipt` (Receipt type from use-receipts)
- Fields: recipient email (pre-filled from `payer_snapshot.email`), custom message
- Receipt summary: receipt number, payer name, amount
- Calls `supabase.functions.invoke('send-receipt-email', ...)`
- Shows success/error toasts

### 4. Add `useSendReceiptEmail` hook to `src/hooks/use-receipts.ts`
- Mutation hook wrapping the edge function call

### 5. Update `src/pages/app/ReceiptDetail.tsx`
- Import `SendReceiptDialog`
- Add "Send Email" button next to "Download PDF" button (Mail icon)
- Add `sendDialogOpen` state + render dialog

## Technical Notes
- The receipt PDF is already generated by `generate-receipt-pdf` — the new edge function will reuse the same pdfmake logic by calling `supabase.functions.invoke('generate-receipt-pdf')` internally via service role, or duplicate the PDF generation inline to avoid the extra network hop. Given the function is self-contained, duplicating the PDF call via the existing function is cleaner.
- Uses existing Brevo API key — no new secrets needed.
- The `getProductionUrl()` pattern from `SendInvoiceDialog` will be reused for verification links.
