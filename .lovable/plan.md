

# Overhaul Email PDF Attachment to Match Download Templates

## Problem

The email attachment PDF is generated using **pdfmake** with a generic layout that barely differentiates between templates. The user's uploaded reference PDFs (generated by `generate-pdf` as HTML) show four distinct designs (Basic, Professional, Modern, Enterprise), but the email attachment ignores most template-specific styling — producing a "watered down" version that doesn't match what users see when downloading.

## Root Cause

In `send-invoice-email/index.ts`, the `generateInvoicePdfBase64` function has template-specific **headers only** (lines 652-765), but uses **shared generic layouts** for billing sections, items table, totals, payment instructions, and footer — regardless of which template was selected.

## Solution

Rewrite `generateInvoicePdfBase64` in `supabase/functions/send-invoice-email/index.ts` so that each template variant produces a fully distinct pdfmake document matching the reference PDFs:

### Template-specific changes needed:

**Basic (minimal):**
- Simple "Invoice INV-XXXX" left-aligned, dates right-aligned
- Thin separator line
- Plain FROM/TO text blocks (no cards, no borders)
- Clean items table, simple totals (Subtotal + Total only)
- Payment instructions in a key-value table layout
- Minimal footer: just issuer name and email

**Professional (standard):**
- Logo or business name top-left, "INVOICE" + number + status badge top-right
- Metadata line (date, due, currency)
- Separator, then BILL TO section alongside "Amount Due" highlight box
- Items table, then "Invoice Summary" box (Total Items, Subtotal, Grand Total, Amount Due)
- Separate right-aligned totals (Subtotal, Total)
- Payment instructions with labeled key-value rows
- QR code + verification footer

**Modern:**
- Full-width colored header bar (fillColor on table cell) with white "INVOICE", number, dates, status
- FROM and TO in bordered card-style boxes (table cells with borders and light background)
- Items table wrapped in a bordered container
- Totals in a bordered card box
- Payment instructions in green-tinted box
- QR + verification footer

**Enterprise:**
- Centered letterhead: colored top border, logo, business name, TIN, address, colored bottom border
- 4-column metadata grid (Invoice No, Date, Due Date, Status badge)
- Thin colored separator
- ISSUER/RECIPIENT two-column section with colored labels
- Items in bordered table
- Clean totals
- Double-border footer with QR + verification

### Implementation approach:

Rather than one monolithic function with shared sections, restructure into a dispatcher:

```typescript
async function generateInvoicePdfBase64(...) {
  const tplHeaderStyle = templateSnapshot?.layout?.header_style || 'standard'
  
  // Each builder returns full pdfmake content array + styles
  switch (tplHeaderStyle) {
    case 'minimal': return buildMinimalPdf(...)
    case 'modern': return buildModernPdf(...)
    case 'enterprise': return buildEnterprisePdf(...)
    default: return buildStandardPdf(...)
  }
}
```

Each builder function constructs the complete `content` array with template-specific styling for ALL sections (header, billing, items, totals, payment, footer), not just the header.

### Key pdfmake techniques for matching designs:
- **Colored header bars**: Table with single cell, `fillColor` set to brand color, `margin` for padding
- **Card-style boxes**: Table cells with explicit `hLineWidth`/`vLineWidth` and `fillColor: '#fafafa'`
- **Status badges**: Inline table with `fillColor` and `noBorders` layout
- **Invoice Summary box**: Bordered table with labeled rows
- **Green payment box**: Table with `fillColor: '#f0fdf4'` and green border
